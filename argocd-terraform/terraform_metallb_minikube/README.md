# terraform_metallb_minikube

Terraform-модуль для развёртывания [MetalLB](https://metallb.universe.tf/) в локальном Kubernetes-кластере (Minikube / kubeadm) через Helm.

## Что делает модуль

1. Устанавливает MetalLB через официальный Helm-чарт в namespace `metallb-system`.
2. Ждёт регистрации CRD (по умолчанию 30 секунд).
3. Создаёт `IPAddressPool` с именем `local-pool` и указанным диапазоном IP.
4. Создаёт `L2Advertisement` для анонса адресов через ARP (Layer 2).

## Переменные


| Переменная         | По умолчанию                      | Описание                          |
| ------------------ | --------------------------------- | --------------------------------- |
| `chart_version`    | `0.15.3`                          | Версия Helm-чарта MetalLB         |
| `ip_range`         | `192.168.100.200-192.168.100.250` | Диапазон IP для пула LoadBalancer |
| `crd_wait_seconds` | `30s`                             | Пауза ожидания регистрации CRD    |


## Outputs


| Output                  | Описание                         |
| ----------------------- | -------------------------------- |
| `metallb_chart_version` | Версия установленного чарта      |
| `metallb_status`        | Статус Helm-релиза               |
| `metallb_ip_range`      | Используемый диапазон IP-адресов |


## Требования

- Провайдеры: `hashicorp/helm >= 3.1.1`, `hashicorp/time >= 0.9.0`, `gavinbunney/kubectl >= 1.14.0`
- Кластер без встроенного балансировщика (bare-metal / Minikube без `tunnel`)

## Подсеть 192.168.100.0/24

Рекомендуемое распределение адресов:


| Диапазон                 | Назначение                     |
| ------------------------ | ------------------------------ |
| `192.168.100.254`          | Шлюз                           |
| `192.168.100.1 – .199`   | Узлы кластера и прочее         |
| `192.168.100.200 – .250` | **Пул MetalLB (LoadBalancer)** |
